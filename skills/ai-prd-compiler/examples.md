# AI-PRD Compiler — examples

## Good AskQuestion (contextual buttons)

**Вопрос 1.** Что происходит с отклонённым skill после модерации?

| Button label |
|--------------|
| Терминальный отказ — автор видит причину в профиле, переотправить нельзя (Recommended) |
| Автор может исправить и отправить тот же skill на повторную модерацию |
| Переотправка запрещена; автор создаёт новую заявку с нуля |

**Вопрос 2.** Нужна ли регистрация для просмотра каталога?

| Button label |
|--------------|
| Каталог, поиск и скачивание — без регистрации (Recommended) |
| Каталог и поиск без регистрации; скачивание — после входа |
| Весь каталог только для авторизованных |

## Bad AskQuestion

**Вопрос 1.** Нужна ли регистрация?

- Да
- Нет

**Вопрос 2.** Какой вариант?

- Вариант A
- Вариант B

## Good markdown fallback

**Вопрос 1.** Кто может ставить emoji-лайк на карточке?

1. **Все посетители** — гости и пользователи без ограничений
2. **Только авторизованные** — гость видит счётчик, но не может поставить реакцию
3. **Никто** — только просмотр агрегированного счётчика без интерактива
